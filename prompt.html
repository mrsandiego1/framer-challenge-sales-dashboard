Build your own live sales challenge dashboard with Claude

A copy-paste prompt for Cowork mode. Answer a few questions, Claude builds a shareable dashboard in minutes. Works for any timed revenue goal: product launch window, freelance sprint, Framer challenge, whatever you're working toward.

1What you end up with

  • An index.html file in your Cowork project folder you can open in any browser.
  • A weekly breakdown with itemized sales, pace vs goal, two charts, milestones, and income by channel.
  • Your business name, logo, and X handle baked into the header.
  • Optionally pushed to a public GitHub Pages URL you can share anywhere.

2Before you start, have ready

  • Your challenge start date (e.g. April 6, 2026).
  • How many days the challenge runs (e.g. 90 days). Claude calculates the end date for you.
  • Your goal amount in dollars.
  • Your business name as you want it in the header.
  • A logo image (PNG or JPG). Drop it into your Cowork project folder and note the filename (e.g. logo.png).
  • Your X/Twitter handle and full URL (e.g. @yourhandle and https://x.com/yourhandle).

3How to use it

  1. Open a new Cowork conversation in the project folder where you want the dashboard.
  2. Drop your logo file into that folder first.
  3. Copy the prompt below and paste it as your first message.
  4. Claude asks follow-up questions to fill in your details, builds the HTML, opens it locally, then offers to push to GitHub.
  5. If you want the public URL, have a GitHub account ready (empty repo is fine). Claude walks you through creating a Personal Access Token if needed.

The prompt

Click Copy, then paste into Cowork as your first message.
I want you to build me a live sales challenge dashboard as a single index.html file in my Cowork project folder. Before writing any code, gather my details, then build the file, then show it to me locally, then offer to deploy it to GitHub Pages. Step 1: Gather my challenge details. Use AskUserQuestion to collect these one at a time (or grouped where it makes sense): - Challenge name (short, e.g. "Framer Challenge" or "Q2 Sprint"). - Start date (accept free-form like "April 6 2026" or "2026-04-06"). - Duration in days (accept a number, e.g. 90). - After I answer start date and days, compute the end date yourself and state it back to me for confirmation before proceeding. - Dollar goal (e.g. $25,000). - Business name as it should appear in the header. - X/Twitter handle with @ (e.g. @yourhandle). - Full X URL (e.g. https://x.com/yourhandle). - Logo filename in the folder (e.g. logo.png). Confirm via bash that the file exists before continuing. If it's missing, ask me to drop it in and tell me the exact folder path. - Optional: income channels I want to track separately (e.g. "hosting commissions, product sales, coaching"). Default to a single "Revenue" channel if I don't care. Step 2: Build the dashboard. Create index.html in the project folder root with: - A circular logo (use my uploaded file as an img tag) in the upper-left header, wrapped in a link to my X URL, on a dark circle with a subtle hover scale. - Business name as the h1 next to the logo, with challenge name, goal, and date range as the subtitle. - A prominent black Twitter badge with the X glyph and my @handle, also linking to my X URL, placed right under the subtitle. - A status pill on the right of the header showing current week number and status. - A 6-card stat grid: earned so far, goal, % complete, remaining, weeks left, needed per week to hit pace. - A progress bar showing progress to goal, with an ideal pace marker at the current week position. - A dark Pace check card with three stats: where ideal linear pace puts me at end of current week, my gap vs ideal pace, and projected total at current run rate. - Two charts using Chart.js from CDN: a line chart of cumulative earnings vs ideal pace, and a bar chart of weekly income (highlight the current week in a different color). - A weekly breakdown table: Week #, Dates, Earned, delta $ WoW, delta % WoW, Cumulative, % of goal, Remaining, Notes. The Notes cell should support a short top-line note plus a sub-list of individual sale line items (each line item = amount + description). Split the duration into 7-day weekly buckets. - An Income by channel card with a bar per channel plus a weekly split sub-table. - A Milestones card with one entry per significant event (launch, first sale, etc.). - Footer line showing last-updated date. - All meta tags: title, description, Open Graph title/description/image (point og:image at the logo filename), Twitter card tags with my @handle. - Responsive styles that stack on screens under 900px. Use Chart.js 4.5.0 from https://cdn.jsdelivr.net/npm/chart.js@4.5.0/dist/chart.umd.js with the integrity hash sha384-iU8HYtnGQ8Cy4zl7gbNMOhsDTTKX02BTXptVP/vqAWIaTfM7isw76iyZCsjL2eVi. Keep all CSS and JS inline so the file is fully self-contained (except for the logo image reference). Use a clean, modern design: off-white background (#f7f7f5), white cards with subtle borders (#e9e9e4), 14px rounded corners, SF/Inter system font stack, green (#0a7d3a) for positive/earned, amber (#b8860b or #a65b00) for current/warning, dark pill backgrounds. Step 3: Show me the file locally. After building, give me a computer:// link to the file so I can open it in my browser. Do not ask me if I want to see it, just give me the link. Step 4: Offer GitHub deployment. After I have opened it locally, ask (using AskUserQuestion) whether I want to push this to a public GitHub repo so I can share it and let it update live. If yes: - Ask me for my GitHub username and the repo name (tell me to create the empty repo on github.com first if I have not). - Ask for a Personal Access Token (walk me through creating one at Settings then Developer settings then Personal access tokens then Fine-grained tokens, scoped to just this repo with Contents read/write). - Initialize a git repo in the folder, create a README and .gitignore, add the logo and index.html, commit on main. - Set the remote URL with the token embedded so future pushes work silently: https://USERNAME:TOKEN@github.com/USERNAME/REPO.git - Push main to origin. - Tell me how to enable GitHub Pages (Settings then Pages then Deploy from branch then main then root) and give me the expected URL: https://USERNAME.github.io/REPO/ Step 5: Teach me how to update it going forward. Tell me that when a new sale comes in, I just tell you the details (amount, what was sold, any relevant notes) and you will edit the weekly items array, recompute totals, commit, and push. Tell me new earnings should always be itemized under their week, not collapsed into a single summary string. Do not skip Step 1. Do not assume defaults for anything I have not told you. Ask, build, show, then deploy.

4Tips after it's built

When a new sale comes in, tell Claude the amount, date, what it was, and any context (e.g. "got $135 today, mid-tier hosting commission, client used a coupon that reduced my cut"). Claude drops it as a line item under the current week, updates totals, and pushes live if you're connected to GitHub.

If the logo looks cropped weird inside the circle, ask Claude to adjust object-fit on .logo-circle img. Try contain instead of cover, or add a little padding.

To change colors, fonts, or add sections later, just describe what you want. The file is set up to be edited conversationally.
